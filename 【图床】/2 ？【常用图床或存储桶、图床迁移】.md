# 图床要求

- 是否稳定
- 免费或付费
- 单个文件大小
- 总容量限制
- 支持的文件格式（图片、视频、压缩文件、文本等）
- 支持批量下载或上传
- 支持远程删除

## ！！！[Cloudflare](https://dash.cloudflare.com/8b2f87d744f1bee78ae72631a22a61f3/home/domains) [R2](https://dash.cloudflare.com/8b2f87d744f1bee78ae72631a22a61f3/r2/plans)：虽然只是存储桶，但稳定快速！免费额度很大！

### 特点

#### 💸 1. **零出站流量费**

Cloudflare R2 最大的优势是 **不收取数据下载（egress）费用**，这在传统对象存储（如 AWS S3）中往往是最贵的一项费用。对于图床这种主要是对外“看图/加载”的场景，这能节省不少成本。

#### 🌍 2. **全球访问加速**

R2 能很好地配合 Cloudflare 的全球 CDN 网络，访问速度快、延迟低。特别是配合设置合理的缓存策略，访问体验可媲美付费 CDN。

#### 🧰 3. **标准 S3 兼容 API**

支持 S3 协议的标准 API，你可以用常见 SDK（例如 aws-sdk、boto3 等）来上传/读取，这对于自动化上传或图床工具对接很方便。

#### 📦 4. **免费额度够用（个人/小流量场景）**

Cloudflare R2 提供每月一定的 **免费存储和请求额度**（如 10GB 存储 + 读写操作免费额度），对于个人博客、小项目图床足够。

#### ❗ 缺点：**读取仍算操作费（Class B）**

虽然 R2 不收数据流量费，但每次从 R2 拉取对象（`GetObject`）会被算作 Class B 操作，如果请求量极大，操作费用可能不小。  
👉 解决策略：借助 **Cloudflare CDN 缓存**（Cache Rules）让热门图片主要从边缘节点返回，这样就不会频繁打到 R2 后端。

#### ❗ 缺点：**自定义域名要求与你 DNS 配置有关**

如果要用自己域名（如 `img.yoursite.com`）访问 R2，通常需将域名托管到 Cloudflare 并正确设置 DNS

### 📌 哪些场景特别适合 R2 做图床？

✅ 个人博客或小型网站静态资源托管  
✅ 技术博客/文档站需要稳定可控的图片服务  
✅ 希望避免高 CDN 流量费用  
✅ 想自己搭建自动上传/管理工具（如结合 PicGo、自动脚本等

# ？？？？？图床开发：[telegraph-Image](https://github.com/x-dr/telegraph-Image?tab=readme-ov-file)（Cloudflare + Github）

1. 无限图片储存数量，你可以上传不限数量的图片；
2. 无需购买服务器，托管于Cloudflare的网络上，当使用量不超过Cloudflare的免费额度时，完全免费；
3. 无需购买域名，可以使用Cloudflare Pages提供的*.pages.dev的免费二级域名，同时也支持绑定自定义域名；
4. 支持图片审查API，可根据需要开启，开启后不良图片将自动屏蔽，不再加载；
5. 支持后台图片管理，日志管理，查看访问前20的Referer、IP、img,可以对上传的图片进行在线预览，添加白名单，黑名单等操作

### 利用Cloudflare pages部署

1. 点击[Use this template](https://github.com/x-dr/telegraph-Image/generate)按钮创建一个新的代码库。    
2. 登录到[Cloudflare](https://dash.cloudflare.com/)控制台.    
3. 在帐户主页中，选择`pages`>  `Create a project` > `Connect to Git`    
4. 选择你创建的项目存储库，在`Set up builds and deployments`部分中，`Framework preset(框架)`选`Next.js`即可。
